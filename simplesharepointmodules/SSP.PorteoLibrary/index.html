Build-PorteoSite

Syntax 

Build-PorteoSite [-tenantUrl] <String> [-credentials] <PSCredential> [-siteName] <String> [-title] <String> [-owner] <String> [[-type] <String>] [[-srcConnection] <Object>] [<CommonParameters>]

Parameters

Name Alias Description Required? Pipeline Input Default Value
-tenantUrl This parameter holds the tenant adminstrative site url, e.g. https://mysite-admin.sharepoint.com true false
-credentials This parameter holds the credentials for the tenant adminstrator. true false
-siteName This parameter holds the name of the site to be created. true false
-title This parameter holds the title for the site to be created. true false
-owner This paramter holds the owner of the new site, which is an email address. true false
-type This paramter holds the type of site to be created, e.g. TeamSite or CommunicationSite. false false TeamSite
-srcConnection This paramter holds the source to copy the template from. If it is not specified, the base *Porteo*
site is built.		 false false

Examples

EXAMPLE 1 
$creds = Get-Credential
Gets the Credential for the subsequent examples.
EXAMPLE 2 
$siteUrl = Build-PorteoSite -TenantUrl "https://mysite-admin.sharepoint.com" -Credentials $creds `
             -SiteName "SiteTemplate" -Title "The Site Template" -Type "TeamSite" `
             -Owner "name@site.onmicrosoft.com"
Builds the first site to use as a template. Now, modify this site adding to lists, pages, documents.
EXAMPLE 3 
$srcConn = Connect-PnPOnline -Url $siteUrl -Credentials $creds
Gets the connection for the template site from which you will copy.
EXAMPLE 4 
$dstCreds = Get-Credential
Gets the tenant credentials to to build the new site.
EXAMPLE 5 
$newSiteUrl = Build-PorteoSite -TenantUrl "https://client-admin.sharepoint.com" -Credentials $dstCreds `
             -SiteName "MySite" -Title "My Great Site" -Type "TeamSite" `
             -Owner "admin@company.com" -SrcConnection $srcConn
Builds the new site "MySite" on the specified tenant and provisions it from the source site.

Build-PorteoSiteFromTemplate

Build-PorteoSite -TenantUrl "https://mysite-admin.sharepoint.com" -Credentials (Get-Credential) ` -SiteName "Site1" -Title "The Site" -Owner "name@mysite.onmicrosoft.com" -Type "TeamSite" ` -SrcConnection $srcConnection

This function builds a *Porteo* site. A *Porteo* site is a ***SimpleSharepoint*** base site, based on
either a **TeamSite** or **ComunicationSite** template type and has the following ***SimpleSharepoint*** lists:

* Site Assets (SiteAssets)
* Documents (Shared Documents)
* Journal
* Requests
* SiteCollectionList
* Links
* SubsiteLists

At this point, you would create this base site, then add contents to the above lists. You may
also add some other lists, add to the _SiteAssets_ and any other *Pages* with content and
branding. This created and modified site will be used to provision other sites on other
tenants.

To provision a new site, you extract a template from the desired site using
Get-SSPSiteTemplate with a file name (.pnp extension)
and now use the `-Template` argument to specify this file and
use the `-TenantUrl` and `-Credentials` arguments to create the site on the target tenant. See
the series of Examples below to see the process and sequence of operations.

Then the target site is provisioned from the site associated with the template file.

The site Pages are copied as well as the Pages' contents. Furthermore, the *Porteo* standard lists,
which are listed above, have their structure and contents copied.

The examples provide in sequence the process by which you may create a base *Porteo* site, modify it,
and then use the result to provision another site.

***Note***: If the tenant site already exists, this function will return with an Error. You must either
delete the target site first, or give use a different name.

Syntax 

Build-PorteoSiteFromTemplate [-template] <String> [-tenantUrl] <String> [-credentials] <PSCredential> [-siteName] <String> [-title] <String> [-owner] <String> [[-type] <String>] [[-colorPaletteUrl] <String>] [[-addInstalledApps] <Boolean>] [<CommonParameters>]

Parameters

Name Alias Description Required? Pipeline Input Default Value
-template This paramter holds the template file. true false
-tenantUrl This parameter holds the tenant adminstrative site url, e.g. https://mysite-admin.sharepoint.com true false
-credentials This parameter holds the credentials for the tenant adminstrator. true false
-siteName This parameter holds the name of the site to be created. true false
-title This parameter holds the title for the site to be created. true false
-owner This paramter holds the owner of the new site, which is an email address. true false
-type This paramter holds the type of site to be created, e.g. TeamSite or CommunicationSite. false false TeamSite
-colorPaletteUrl This parameter holds the color palette url for the theme, which should be in the template file. false false
-addInstalledApps This parameter adds any installed apps on the tenant to the site. false false False

Examples

EXAMPLE 1 
$dstCreds = Get-Credential
Gets the tenant credentials to to build the new site.
EXAMPLE 2 
$newSiteUrl = Build-PorteoSiteFromTemplate -Template /tmp/template.pnp
             -TenantUrl "https://client-admin.sharepoint.com" -Credentials $dstCreds `
             -SiteName "new-site" -Title "The New Site" -Type "TeamSite" `
             -Owner "admin@company.com"
Builds the new site "New Site" at https://client.sharepoint.com/sites/new-site and provisions it from the template file.
EXAMPLE 3 
$colorPaletteUrl = Add-SSPThemeToTemplate -Path /tmp/template.pnp -Connection $srcConn
$newSiteUrl = Build-PorteoSiteFromTemplate -Template /tmp/template.pnp
             -TenantUrl "https://client-admin.sharepoint.com" -Credentials $dstCreds `
             -SiteName "new-site" -Title "The New Site" -Type "TeamSite" `
             -Owner "admin@company.com" -ColorPaletteUrl $colorPaletteUrl -AddInstalledApps $true
Builds the new site "New Site" at https://client.sharepoint.com/sites/new-site and provisions it from the template file using the current theme of the source site that was put into the template file.

Build-PorteoSiteListFromTemplate

Syntax 

Build-PorteoSiteListFromTemplate [-template] <String> [-tenantUrl] <String> [-credentials] <PSCredential> [-siteList] <List`1> [-owner] <String> [[-type] <String>] [[-colorPaletteUrl] <String>] [[-addInstalledApps] <Boolean>] [<CommonParameters>]

Parameters

Name Alias Description Required? Pipeline Input Default Value
-template This paramter holds the template file. It must be a ".pnp" file name. true false
-tenantUrl This parameter holds the tenant adminstrative site url, e.g. https://mysite-admin.sharepoint.com true false
-credentials This parameter holds the credentials for the tenant adminstrator that can create sites. true false
-siteList This parameter holds the urls and titles of the sites to be created. It is a list of site specs.
A site spec is a list of url and site title.		 true false
-owner This paramter holds the owner of the new site, which is an email address. true false
-type This paramter holds the type of site to be created, e.g. TeamSite or CommunicationSite.
This parameter MUST be the same type as the template.		 false false TeamSite
-colorPaletteUrl This parameter holds the color palette url for the theme, which should be in the template file. false false
-addInstalledApps This parameter adds any installed apps on the tenant to the site. false false False

Examples

EXAMPLE 1 
$dstCreds = Get-Credential
Gets the tenant credentials to to build the new site.
EXAMPLE 2 
$srcCreds = Get-Credential
Gets the tenant credentials for the source site template.
EXAMPLE 3 
$srcConn = Connect-PnPOnline -Url https://company.sharepoint.com/sites/site-template -Credentials $srcCreds -ReturnConnection
Gets the authenticated connection for the template site.
EXAMPLE 4 
Get-SSPSiteTemplate -FileName /tmp/template.pnp -SrcConnection $srcConn
Gets and writes the template for the source site to the file /tmp/template.pnp.
EXAMPLE 5 
$colorPaletteUrl = Add-SSPThemeToTemplate -Path /tmp/template.pnp
Adds the current Theme to the template file. (THIS IS A WORKAROUND FOR BROKEN Theme Handler)
EXAMPLE 6 
$newSiteUrl = Build-PorteoSiteListFromTemplate -Template /tmp/template.pnp
             -TenantUrl "https://client-admin.sharepoint.com" -Credentials $dstCreds `
             -SiteList @(@("site1", "Site 1 Title"), @("site2", "Site 2 Title")) -Type "TeamSite" `
             -Owner "admin@company.com" -ColorPaletteUrl $colorPaletteUrl -AddInstalledApps
Builds the new sites "Site 1 Title" and "Site 2 Title" at https://client.sharepoint.com/sites/site1 and https://client.sharepoint.com/sites/site2 respectively, and provisions them from the template file /tmp/template.pnp.

Copy-PorteoFolder

Syntax 

Copy-PorteoFolder [[-folderName] <Object>] [[-srcConnection] <Object>] [[-destConnection] <Object>] [<CommonParameters>]

Parameters

Name Alias Description Required? Pipeline Input Default Value
-folderName false false
-srcConnection false false (Get-PnPConnection)
-destConnection false false (Get-PnPConnection)

Examples

EXAMPLE 1 
$srcSite = "https://client1.sharepoint.com/sites/BaseSite"
$dstSite = "https://client2.sharepoint.com/sites/NewSite"
$srcCreds = Get-Credential
$dstCreds = Get-Credential
$srcConn = Connect-PnPOnline -Url $srcSite -Credentials $srcCreds
$dstConn = Connect-PnPOnline -Url $dstSite -Credentials $dstCreds
Copy-PorteoFolder -FolderName "NameOfFolder" -SrcConnection $srcConn -DestConnection $dstConn
