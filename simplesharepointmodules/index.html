SimpleSharepoint Modules

The following Powershell modules facilitate the provisioning of new sites on SimpleSharepoint's clients' tenants. The functionality is provided using two different modules, SSP.PorteoLibrary and SSP.PnPLibrary. The main functionality to be used is in the Build-PorteoSite function provided by the SSP.PorteoLibrary module. The Build-PorteoSite has two functions.

Build a base Porteo site
This function builds a base Porteo site with standard SimpleSharepoint lists. See the function for details on which lists are created. Then you would add content to this site to serve as provisioning new sites.
Copy and Provision a new Porteo Site from a source site.
This function will create a new site on a target tenant and copy the source site lists, pages, page contents, and site assets from the base site. Typically what you would do is create a base site using the above function first. Then, you would add content to that site, and once that is done and can be used to provision a new clients site, then you use this functionality to provision the newly created site.

As a preliminary example, the following Powershell commands will create a base PorteoSite and provision a new one from it.


						$creds = Get-Credential
						$siteUrl = Build-PorteoSite -TenantUrl "https://mysite-admin.sharepoint.com" -Credentials $creds `
									 -SiteName "SiteTemplate" -Title "The Site Template" -Type "TeamSite" `
									 -Owner "name@site.onmicrosoft.com"
						$srcConn = Connect-PnPOnline -Url $siteUrl -Credentials $creds
						$dstCreds = Get-Credential
						$newSiteUrl = Build-PorteoSite -TenantUrl "https://client-admin.sharepoint.com" -Credentials $dstCreds `
									 -SiteName "MySite" -Title "My Great Site" -Type "TeamSite" `
									 -Owner "admin@company.com" -SrcConnection $srcConn

The documentation for the modules and their functions may be accessed from the following links.

SSP.PorteoLibrary

This library contains functions to provision common SimpleSharepoint sites. The most notable function is be Build-PorteoSite

SSP.PnPLibrary

This library contains more primitive functions to provision sites and other things on a Sharepoint tenant. The functions in this module are used by the SSP.PorteoLibrary to create common configured sites for a SimpleSharepoint tenant.

You may want to use these functions if you want to do more than the Build-PorteoSite function provides

Creating Multiple Sites from a Site Template

This example illustrates creating the template file for an existing site and then replicating it using the functions Build-PorteoSiteListFromTemplate 

#
# Emanuel Group ArrayList of sites you want to create using url and title pairs.
#
$sites = @(
    @("tls", "Top Level Site"),
    @("emanuel-health", "Emanuel Health"),
    @("prs-global", "PRS Global"),
    @("epic-consulting", "Epic Consulting"),
    @("t1-construction", "T1 Construction"),
    @("phenix-salon", "Phenix Salon"),
    @("manna-foundation", "Manna Foundation"),
    @("resource-center", "Resource Center"),

    @("emanuel-health", "Emanuel Health"),
    @("emanuel-health/beaver-dam-nursing-and-rehab", "Beaver Dam Nursing and Rehab"),
    @("emanuel-health/the-cottages-of-clayton", "The Cottages of Clayton"),
    @("emanuel-health/elizabeth-care-center", "Elizabeth Care Center"),
    @("emanuel-health/harbor-healthcare-of-ironton", "Harbor Healthcare of Ironton"),
    @("emanuel-health/madison-park", "Madison Park"),
    @("emanuel-health/mountain-manor-of-paintsville", "Mountain Manor of Paintsville"),
    @("emanuel-health/emanuel-pointe", "Emanuel Pointe")
)

### Get/Create the Source Template File
$srcSite = "https://theemanuelgroups0.sharepoint.com/sites/Department-template"
$srcEmail = "edg@simplesharepoint.com"
$srcPlainPassword = "great_password"

$srcSecurePassword = $srcPlainPassword | ConvertTo-SecureString -AsPlainText -Force
$srcCreds = New-Object System.Management.Automation.PSCredential ($srcEmail, $srcSecurePassword)

$srcConn = Connect-PnPOnline -Url $srcSite  -Credentials $srcCreds -ReturnConnection

# Note, Windows may need another place for this file, use "C:\somwhere\template.pnp". Make sure it ends in ".pnp"
$template = Get-SSPSiteTemplate -FileName "/tmp/template.pnp" -SrcConnection $srcConn

## Now create each site listed above in the tennant from the template.
## Note: this admin site may be completely different, different credentials, etc.
$admSite = "https://theemanuelgroups0-admin.sharepoint.com"
$owner = "edg@simplesharepoint.com"
$PlainPassword = "great_password"

$SecurePassword = $PlainPassword | ConvertTo-SecureString -AsPlainText -Force
$dstCreds = New-Object System.Management.Automation.PSCredential ($owner, $SecurePassword)

Build-PorteoSiteListFromTemplate -Template $template -SiteList $sites -TenantUrl $admSite -Credentials $dstCreds `
             -Owner $owner -Type "TeamSite"
